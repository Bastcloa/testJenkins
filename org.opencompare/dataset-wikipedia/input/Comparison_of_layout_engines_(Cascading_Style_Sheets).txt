===Trident grammar and rule notes===
# <tt id="trident-important-bug">!important</tt> &mdash; Prior to 7.0, <code>!important</code> doesn't override rules defined later in the same declaration block.
# <tt id="trident-import-bug">@import</tt> &mdash; Prior to 8.0, media type support is broken. For <code>@import &lt;URL&gt; &lt;MEDIA&gt;</code>, IE requests as the URL "<code>&lt;URL&gt; &lt;MEDIA&gt;</code>" and will include the <code>url()<code> token and/or quotes in the URL, too, if present. Cannot import more than 35 stylesheets.

===Presto grammar and rule notes===
# <tt id="presto_at-import">@import</tt> &mdash; Whilst Gecko, WebKit and iCab download all media stylesheets immediately, Opera only downloads handheld, print, projection and screen media, as well as speech if "voice" feature is enabled and TV, on TV devices. Text browser emulation mode is only a user stylesheet, so it does not switch to tty media type. This is consistent with older text browsers, which do not respect any CSS.

=== General selector notes ===
# <tt id="general_rw">:read-only</tt> and <tt>:read-write</tt> &mdash; Both Presto and KHTML handle the case of the <code>contenteditable</code> attribute incorrectly.
# <tt id="general_selection">::selection</tt> has been removed from the CSS3 specification, but is still supported by many browsers.

=== Trident selector notes ===
# <tt id="trident_active">:active</tt> &mdash; Prior to 8.0, <code>:active</code> is only supported on anchor elements.
# <tt id="trident_hover">:hover</tt> &mdash; Prior to 7.0, <code>:hover</code> is only supported on anchor elements.
# <tt id="trident_class">.one.two</tt> &mdash; Prior to 7.0, only <code>.two</code> class selector was taken into consideration.
# <tt id="trident_universal">*</tt> &mdash; Prior to 7.0 this was treated as a single or no element.
# <tt id="trident_attr">[attr]</tt> &mdash; Matches every <code>td</code> and <code>th</code> in a table when the attribute is <code>colspan</code> (regardless of whether any actually have a <code>colspan</code> attribute).<ref group="t">{{citation |url=https://connect.microsoft.com/IE/feedback/ViewFeedback.aspx?FeedbackID=446817 |title=Attribute selector (which matches the attribute, 'colspan' for TD and TH elements) matches every TD and TH element |last=Hopkins |first=James}}</ref> This may not actually be a bug, as ambiguity exists in the specification.<ref group="t">{{citation |url=http://www.webdevout.net/tidings/2009/03/23/ie-8-css-21-support-results/#comment-4168 |title=Comment on "IE 8 CSS 2.1 support results" |last=Hammond |first=David}}</ref>
# <tt id="trident_first-letter">:first-letter</tt>, <tt>:first-line</tt> &mdash; Prior to IE9 in 6.0, combining <code>:first-letter</code> rules with others may be problematic.<ref group="t">{{citation |url=http://haslayout.net/css/-first-letter-Ignore-Bug |title=:first-letter Ignore Bug |publisher=hasLayout.net}}</ref> In 8.0, rules with <code>!important</code> are ignored inside <code>:first-line</code> and <code>:first-letter</code> declarations.<ref group="t">{{citation |url=https://connect.microsoft.com/IE/feedback/details/478138/declaration-which-includes-important-keyword-is-ignored-when-used-within-a-first-letter-or-first-line-rule |title=Declaration which includes !important keyword is ignored when used within a :first-letter or :first-line rule |last=Hopkins |first=James}}</ref>

=== Gecko selector notes ===
# <tt id="gecko_gencon">(:):before, (:):after</tt> &mdash; CSS2.0 behavior : some properties are unimplemented prior to 1.9.1.<ref group="g">{{citation |url=https://bugzilla.mozilla.org/show_bug.cgi?id=237119 |title=Bug 237119 - property 'display' is ignored for generated content |publisher=Mozilla}}</ref>

=== WebKit selector notes ===
# <tt id="webcore_lang">:lang()</tt> &mdash; Only detected when explicitly present on element being tested, attribute not inherited.
# <tt id="webcore_first-line">(:):first-line</tt> &mdash; <code>text-transform</code> doesn't apply with this pseudo-element.<ref group="w">{{citation |url=https://bugs.webkit.org/show_bug.cgi?id=3409 |title=Bug 3409 - CSS1: Safari ignores "text-transform" attribute in "first-line" CSS rules |publisher=Webkit}}</ref>
# <code id="webcore_pseudo-elements">(:):before/after</code> &mdash; some styles can't be applied to <code>:before</code> and <code>:after</code> pseudo-elements, such as animations and transitions.<ref group="w">{{citation |url=https://bugs.webkit.org/show_bug.cgi?id=23209 |title=Bug 23209 - &#91;RFE&#93; CSS Generated Content does not transition |publisher=Webkit}}</ref>

=== Presto selector notes ===
# <tt id="presto_target">:target</tt> &mdash; Prior to 2.5, styles aren't applied when navigating using back and forward buttons.

=== Trident property notes ===
# <tt id="trident_margin">margin</tt> &mdash; <code>inherit</code> inherits the computation and recomputes it instead of inheriting the computed value.<ref group="t">{{citation |url=https://connect.microsoft.com/IE/feedback/ViewFeedback.aspx?FeedbackID=344787 |title=Inherited margin with inherit keyword: the computed value's parent should be inherited |last=Talbot}}</ref>
# <tt id="trident_display">display</tt> &mdash; Prior to 7.0, only <code>none</code>, <code>block</code>, <code>inline</code>, <code>table-header-group</code>, and <code>table-footer-group</code> are fully supported. Prior to 8.0, <code>table</code> are not supported, while <code>inline-block</code> is only supported on elements that are naturally inline.
# <tt id="trident_overflow">overflow</tt> &mdash; Prior to 7.0, <code>overflow: visible;</code> is incorrectly supported. In 8.0, <code>scroll</code> makes the element's height equal to its <code>max-height</code> even if the content isn't that tall.<ref group="t">{{citation |url=https://connect.microsoft.com/IE/feedback/ViewFeedback.aspx?FeedbackID=408759 |title=Overflow: scroll causes element to always have its max-height |last=Groot |first=Sven}}</ref> This has been fixed in IE9.
# <tt id="trident_visibility">visibility</tt> &mdash; Prior to 8.0, <code>visibility: collapse;</code> is not supported. In 8.0, inline elements with <code>visibility: visible;</code> inside block elements with <code>visibility: hidden;</code> are not visible.<ref group="t">{{citation |url=https://connect.microsoft.com/IE/feedback/ViewFeedback.aspx?FeedbackID=442805 |title=visibility:visible applied to inline element doesn’t override inherited visibility:hidden value applied to it’s block-level parent |last=Hopkins |first=James}}</ref>
# <tt id="trident_content">content</tt> &mdash; In 8.0, computed <code>attr()</code> values are not updated when the attribute changes.<ref group="t">{{citation |url=https://connect.microsoft.com/IE/feedback/ViewFeedback.aspx?FeedbackID=434925 |title=Content: attr(x) isn't updated when the attribute's value change}}</ref>
# <tt id="trident_border-color">border-color</tt> &mdash; Prior to 7.0, <code>transparent</code> is not supported.
# <tt id="trident_border-style">border-style</tt> &mdash; Prior to 8.0, <code>hidden</code> is not supported.
# <tt>border-style</tt> &mdash; Prior to 7.0, <code>dotted</code> is rendered as <code>dashed</code>.
# <tt id="trident_box-shadow">box-shadow</tt> &mdash; Trident 9.0 renders box-shadow blur value at about half the declared value. Prior to 9.0, trident supports similar functionality since 5.5 using the proprietary Shadow<ref group="t">{{citation |url=http://msdn.microsoft.com/en-us/library/ms533086(VS.85).aspx |title=Shadow Filter |publisher=Microsoft}}</ref> and DropShadow<ref group="t">{{citation |url=http://msdn.microsoft.com/en-us/library/ms532985(VS.85).aspx |title=DropShadow Filter |publisher=Microsoft}}</ref> filters.
# <tt id="trident_position">position</tt> &mdash; Prior to 7.0, fixed positioning was not supported. 7.0 and later support it in standards-compliant mode only.
# <tt id="trident_z-index">z-index</tt> &mdash; Prior to 8.0, <code>z-index</code> is only partially supported.<ref group="t">{{citation |url=http://msdn.microsoft.com/en-us/library/cc351024(VS.85).aspx#positioning |title=Positioning |publisher=}}</ref> In 8.0, floating point values are accepted in addition to integers.<ref group="t">{{citation |url=https://connect.microsoft.com/IE/feedback/details/386914/illegal-parsing-of-a-z-index-decimal-value-instead-of-an-integer |title=Illegal parsing of a z-index decimal value, instead of an integer |last=Hopkins |first=James}}</ref>
# <tt id="trident_list-style-type">list-style-type</tt> &mdash; Prior to 8.0, <code>armenian</code>, <code>decimal-leading-zero</code>, <code>georgian</code>, <code>lower-greek</code>, <code>lower-latin</code>, <code>upper-latin</code> are not supported.
# <tt id="trident_opacity">opacity</tt> &mdash; Prior to 9.0, Trident supported a proprietary alternative.<ref group="t">{{citation |url=http://msdn.microsoft.com/en-us/library/ms532967(VS.85).aspx |title=Alpha Filter |publisher=Microsoft}}</ref>
# <tt id="trident_background-image">background-image</tt> &mdash;  Prior to 8.0, background images are badly positioned in some cases.<ref group="t">{{citation |url=http://css-class.com/test/bugs/ie/escaping-background-image-bug1.htm |title=IE7-/Escaping Background Image Bug - Demo 1}}</ref><ref group="t">{{citation |url=http://css-class.com/test/bugs/ie/escaping-background-image-bug2.htm |title=IE7-/Escaping Background Image Bug - Demo 2}}</ref>
# <tt id="trident_background-attachment">background-attachment</tt> &mdash; Prior to 7.0, <code>fixed</code> was allowed on the <code>body</code> element only.
# <tt id="trident_background-position">background-position</tt> &mdash; Prior to 8.0, fixed positioning is not supported.
# <tt id="trident_font-weight">font-weight</tt> &mdash; Prior to 8.0, incorrect rendering when value is 600.<ref group="t">{{citation |url=http://www.quirksmode.org/css/tests/iewin_fontweight.html |title=IE Windows and Opera - font-weight: 600 vs. bold |last=Koch |first=Peter-Paul |publisher=QuirksMode}}</ref>
# <tt id="trident_text-align">text-align</tt> &mdash; In 8.0, <code>text-align</code> isn't inherited by <code>:before</code> and <code>:after</code> pseudo-elements.<ref group="t">{{citation |url=https://connect.microsoft.com/IE/feedback/ViewFeedback.aspx?FeedbackID=454985 |title='text-align' value isn't inherited by ':before' and ':after' pseudo elements |last=Hopkins |first=James}}</ref>
# <tt id="trident_white-space">white-space</tt> &mdash; Prior to 6.0, <code>pre</code> is not supported. Prior to 8.0, <code>white-space</code> is only partially supported; <code>pre-line</code> and <code>pre-wrap</code> are not supported.<ref group="t">{{citation |url=http://msdn.microsoft.com/en-us/library/cc351024#font |title=Font and Text |publisher=Microsoft}}</ref>
# <tt id="trident_cursor">cursor</tt> &mdash; Does not fail on non-prefixed vendor extensions.
# <tt id="trident_flexbox">flex</tt> &mdash; Experimental in 10.0<ref group="t">{{citation |url=http://msdn.microsoft.com/en-us/library/ie/hh673531(v=vs.85).aspx|title=Flexible Box ("Flexbox") Layout|publisher=Mircosoft}}</ref>
# <tt id="trident_region-break">break-after; break-before; break-inside</tt> &mdash; Though 10.0 does support break-after, break-before, and break-inside for columns, they do not appear to support the properties "region" and "avoid-region".

=== Gecko property notes ===
# <tt id="gecko_display">display</tt> &mdash; Values except inline-block and inline-table supported prior to 1.9.
# <tt id="gecko_z-index">z-index</tt> &mdash; Negatives values are badly supported prior to 1.9.
# <tt id="gecko_content">content</tt> &mdash; <code>none</code> value is unsupported prior to 1.9. This property also fails on any normal (non-pseudo-) elements, which it must support according to the CSS3 "Generated and Replaced Content Module" spec.
# <tt id="gecko_background-position">background-position</tt> &mdash; Versions prior to 1.7 implement the CSS2 syntax, not the proposed CSS2.1 expanded syntax.
# <tt id="gecko_font-size-adjust">font-size-adjust</tt> &mdash; Prior to 1.9, supported on Windows only.
# <tt id="gecko_font-weight">font-weight</tt> &mdash; Only Regular and Bold weights get used, even if Light or Heavy/Black faces are installed, unless using DirectWrite in Gecko 2.0 on Windows 7 or Windows Vista.
# <tt id="gecko_white-space">white-space</tt> &mdash; <code>pre-line</code> is not supported prior to 1.9.1. Prior to 1.9, <code>pre-wrap</code> was only supported experimentally as <code>-moz-pre-wrap</code>.
# <tt id="gecko_visibility">visibility</tt> &mdash; <code>collapse</code> is unsupported prior to 1.8.
# <tt id="gecko_border-radius">border-radius</tt> &mdash; Prior to 1.9.1, border curves are circular, not elliptical as specified by the current CSS3 draft. Short-cut definitions for <code>border-radius</code> read "tl tr br bl" instead of the W3C's "tr br bl tl". When the border style is dotted or dashed, curves are rendered as solid instead.<ref group="g">{{citation |url=https://bugzilla.mozilla.org/show_bug.cgi?id=382721 |title=Bug 382721 - Dotted/dashed -moz-border-radiused corners are rendered as solid |publisher=Mozilla}}</ref>
# <tt id="gecko_page-break-before-after">page-break-before; page-break-after</tt> &mdash; Only the <code>always</code> and <code>auto</code> values are supported.

=== WebKit property notes ===
# <tt id="webcore_max-width">max-width; max-height; min-width; min-height</tt> &mdash; Are not supported on tables.(as commented on 2012-12-18, it is fixed)<ref group="w">{{citation |url=https://bugs.webkit.org/show_bug.cgi?id=25016 |title=Bug 25016 - max-width and max-height are not overriding the width and height properties when 'display' set to 'table'|publisher=Webkit}}</ref>
# <tt id="webcore_font">font</tt> &mdash; The system font keywords that allow designers to tailor presentation to the user's operating system environment are unsupported.
# <tt id="webcore_font-weight">font-weight</tt> &mdash; Only Regular and Bold weights get used, even if Light or Heavy/Black faces are installed.
# <tt id="webcore_page-break-before-after">page-break-before; page-break-after</tt> &mdash; Only the <code>always</code> and <code>auto</code> values are supported.
# <tt id="webcore_white-space">white-space</tt> &mdash; Prior to 522, <code>pre-line</code> and <code>pre-wrap</code> are unsupported.
# <tt id="webcore_visibility">visibility</tt> &mdash; <code>collapse</code> is unsupported prior to 522. Its implementation has the same effect as <code>hidden</code> and is therefore not compliant.<ref group="w">{{citation |url=https://bugs.webkit.org/show_bug.cgi?id=8735 |title=Bug 8735 - CSS 2.1 visibility: collapse rendered as visibility: hidden |publisher=Webkit}}</ref>
# <tt id="webkit_content">content</tt> &mdash; The <code>none</code>, <code>open-quote</code>, <code>close-quote</code>, <code>no-open-quote</code>, <code>no-close-quote</code> and <code>normal</code> values are unsupported. This property also fails on any normal (non-pseudo-) elements, which it must support according to the CSS3 "Generated and Replaced Content Module" spec.
# <tt>font-size</tt> &mdash; The <code>font-size</code> property does not always accept the value 0, in "font-size: 0px" the text is still visible.

=== KHTML property notes ===
# <tt id="khtml_overflow">overflow</tt> &mdash; Values <code>scroll</code> and <code>auto</code> are unsupported.
# <tt id="khtml_page-break-before-after">page-break-before; page-break-after</tt> &mdash; Before 3.5 only the <code>always</code> and <code>auto</code> values were supported.
# <tt id="khtml_visibility">visibility</tt> &mdash; All properties are supported, but the implementation of <code>collapse</code> has the same effect as <code>hidden</code> and is therefore not compliant.

=== Presto property notes ===
# <tt id="presto_counter">counter-increment, counter-reset</tt> &mdash; Implemented the algorithm in REC CSS2.
# <tt id="presto_background-position">background-position</tt> &mdash; Presto versions prior to Opera 8.0 implement the CSS2 syntax, not the proposed CSS2.1 expanded syntax.
# <tt id="presto_font-weight">font-weight</tt> &mdash; Incorrect rendering when value is 600. Renders the text in a compatibility mode for websites made for Trident.
# <tt id="presto_visibility">visibility</tt> &mdash; Prior to 2.5, the value <code>collapse</code> was unsupported in table columns. On rows, it had the same effect as <code>hidden</code> and was therefore not compliant.
# <tt id="presto_cursor">cursor</tt> &mdash; <code>cursor</code> is ignored with dynamic pseudo-classes and custom cursors are unsupported.
# <tt id="presto_content">content</tt> &mdash; Prior to 2.7, the <code>none</code> value was unsupported.<ref group="p">{{citation |url=http://www.quirksmode.org/css/beforeafter_content.html |title=:before/:after and content |publisher=Quirksmode |first=Peter-Paul |last=Koch}}</ref>
# <tt id="presto_font">font</tt> &mdash; <code>inherit</code> should not be allowed together with a font-size value. This should not be parsed, but it's currently done in Opera.

===Prince XML property notes===
# <tt id="prince-visibility">visibility</tt> &mdash; The value <code>collapse</code> is unsupported.<ref group="r" name="prince">{{citation |url=http://www.princexml.com/doc/7.0/css21/ |title=Prince: Unsupported features in CSS 2.1 |publisher=YesLogic}}</ref>
# <tt id="prince-list-style">list-style</tt> &mdash; The values <code>armenian</code> and <code>georgian</code> are unsupported.<ref group="r" name="prince"/>
# <tt id="prince-content">content</tt> &mdash; The values <code>open-quote</code> and <code>close-quote</code> are unsupported.<ref group="r" name="prince"/>

===General value and unit notes===
# <tt id="general_transparent">transparent</tt> &mdash; <blockquote cite="http://www.w3.org/TR/2008/WD-css3-color-20080721/#transparent">CSS1 introduced the ‘transparent’ value for the background-color property. CSS2 allowed border-color to also accept the ‘transparent’ value. The Open eBook(tm) Publication Structure 1.0.1 [OEB101] extended the ‘color’ property to also accept the ‘transparent’ keyword. CSS3 extends the color value to include the ‘transparent’ keyword to allow its use with all properties that accept a <color> value. This simplifies the definition of those properties in CSS3.</blockquote>
# <tt id="general_angle">&lt;angle&gt;</tt> &mdash; The <code>turn</code> unit is unsupported.

===Trident value and unit notes===
# <tt id="trident_rect">rect()</tt> &mdash; Prior to 8.0, <code>rect()</code> was not supported with the correct syntax using commas.
# <tt id="trident_auto">auto</tt> &mdash; In [[quirks mode]] (IE5 emulation mode), <code>auto</code> does not work for <code>margin</code>s, except table elements.
# <tt id="trident_transparent">transparent</tt> &mdash; In 7.0 and 8.0, using <code>transparent</code> in the <code>color</code> property will render the text as black.
# <tt>transparent</tt> &mdash; Prior to 7.0, <code>transparent</code> is not supported on borders (shows as solid black) and is ignored on PNG images.

===Gecko value and unit notes===
# <tt id="gecko_ch">&lt;ch&gt;</tt> &mdash; Prior to version 1.9.1, used the width of the "M" glyph instead of the width of the "0" glyph.<ref group="g">{{citation |url=https://bugzilla.mozilla.org/show_bug.cgi?id=282126 |title=Bug 282126 - What to do about the 'ch' length unit? (Mozilla vendor specific) |publisher=Mozilla}}</ref>

===Presto value and unit notes===
# <tt id="presto_float">&lt;number&gt;</tt> &mdash; Prior to 2.1, there existed a [[Noncommutative geometry|quantization]] error for values greater than 20.47 (not limited to <tt>em</tt>, try any non-pixel units).<ref group="p">{{citation |url=http://www.brunildo.org/test/emarg.pl  |title=Quantization error for em values in Opera 7–9.2 |archiveurl = http://web.archive.org/web/20080321205550/http://www.brunildo.org/test/emarg.pl |archivedate = 2008-03-21}}</ref>
# <tt id="presto_transparent">transparent</tt> &mdash; Keyword is ignored when used with the <tt>outline-color</tt> property. Prior to version 2.2, it was also ignored when used with the <tt>color</tt> and <tt>text-shadow</tt> properties.

==General notes==

===Trident general notes===
# <code id="trident_-ms-">-ms-</code> &mdash; All experimental properties are prefixed with "-ms-", e.g. <code>-ms-interpolation-mode</code> instead of <code>interpolation-mode</code>.

===Gecko general notes===
# <code id="gecko_-moz-">-moz-</code> &mdash; All experimental selectors, properties and values are prefixed with "-moz-", e.g. <code>::-moz-selection</code> instead of <code>::selection</code>.

===WebKit general notes===
# <code id="webcore_-webkit-">-webkit-</code> &mdash; All experimental selectors, properties and values are prefixed with "-webkit-", e.g. <code>-webkit-box-shadow</code> instead of <code>box-shadow</code>.

=== KHTML general notes ===
# <code id="khtml_-khtml-">-khtml-</code> &mdash; All experimental selectors, properties and values are prefixed with "-khtml-", e.g. <code>-khtml-opacity</code> instead of <code>opacity</code>.

===Presto general notes===
# <code id="presto_-xv-">-xv-</code> &mdash; All new selectors, properties and values introduced by CSS3 Speech Module are prefixed with "-xv-" (but not found in CSS2 aural style sheets), e.g. <code>-xv-voice-rate</code> instead of <code>voice-rate</code>.
# <code id="presto_-o-">-o-</code> &mdash; All experimental properties are prefixed with "-o-", e.g. <code>-o-transition-property</code> instead of <code>transition-property</code>.

===DOM equivalents to vendor-specific styles===
#  The DOM properties corresponding to vendor-specific experimental CSS properties are prefixed with the vendor-prefix, without any hyphens, and with first letters capitalised, e.g. <code>element.style.MozBorderRadius</code> corresponding to the <code>-moz-border-radius</code> property, or <code>element.style.OTransform</code> corresponding to the <code>-o-transform</code> property. The exception is Internet Explorer which uses a lower-case prefix, e.g. <code>element.style.msTransform</code>.
